## ğŸ“‹ Background

### Current Issues

1. **Hierarchical numbering** (0.1, 0.2) causes confusion:
   - Sprint 0.10 sorts after 0.2 in file systems
   - Sub-sprints (0.9.1-0.9.4) blur the line between sprints and tasks
   - Not aligned with agile best practices

2. **Industry standard** (from research):
   - Sprints numbered continuously: 1, 2, 3, 4... 10, 11, 12
   - Never reset numbering across phases
   - File names use zero-padding for sorting: `sprint-01.md`, `sprint-10.md`
   - Content uses natural numbers: "Sprint 1", "Sprint 10"

### Research Summary

- âœ… **Scrum/SAFe/Agile frameworks**: Use continuous numbering, no zero-padding in content
- âœ… **Industry practice**: Phases are organizational groupings, not part of sprint numbers
- âœ… **File system workaround**: Zero-pad filenames only, not content
- âœ… **Sub-sprints**: Should be "tasks" within a sprint, not separate sprints

**Reference**: [ADR 002 - Imperative Migrations](../../decisions/002-imperative-migrations.md) research on best practices
